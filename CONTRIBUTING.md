# Contributing to ThesisGuard AI

Thanks for your interest in contributing! Contributions are welcome via issues and pull requests.

Guidelines:

- Search existing issues before creating a new one.
- For bug reports, include steps to reproduce and environment details.
- For feature requests, describe the user benefit and a proposed approach.
- Keep PRs small and focused; one logical change per PR.
- Include tests for new behavior when practical and run `pytest` locally.

Code style:

- Follow existing project style (PEP8 for Python).
- Run `black` or your preferred formatter before creating a PR.

Development:

- Create a topic branch: `git checkout -b feat/my-feature`
- Run tests: `python -m pytest`
- Push branch and open a pull request against `main`.

Maintainers will review PRs and provide feedback. By contributing, you agree that your contributions are licensed under the project's MIT License.
